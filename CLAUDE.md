## Overview

This is a Terraform provider for OpenWebUI that enables infrastructure-as-code management of OpenWebUI resources. The provider is built using the Terraform Plugin Framework (not SDKv2) and manages users, groups, knowledge bases, and models.

## Development Commands

### Building
```bash
go build                                    # Build the provider binary
go build -o terraform-provider-openwebui   # Build with specific output name
```

### Testing
```bash
go test ./...                              # Run all tests
go test -v ./internal/provider/...         # Run provider tests with verbose output
TF_ACC=1 go test -v ./internal/provider/   # Run acceptance tests (requires running OpenWebUI instance)
```

### Documentation Generation
```bash
cd tools
go generate                                # Generate provider documentation, format examples, and update copyright headers
```

### Local Development
To test the provider locally:
1. Build: `go build`
2. Install to local Terraform plugins directory: `~/.terraform.d/plugins/`
3. Configure provider in your Terraform code with required environment variables:
   - `OPENWEBUI_ENDPOINT` - OpenWebUI API endpoint URL
   - `OPENWEBUI_TOKEN` - API authentication token

## Architecture

### Provider Structure

The codebase follows Terraform Plugin Framework patterns with a clear separation of concerns:

```
internal/provider/
├── provider.go              # Main provider configuration, creates all clients
├── *_resource.go           # Resource implementations (groups, knowledge, models)
├── *_data_source.go        # Data source implementations (groups, knowledge, models, users)
└── client/                 # API client implementations
    ├── groups/
    │   ├── client.go       # HTTP client for groups API
    │   └── types.go        # Group data structures
    ├── knowledge/
    ├── models/
    └── users/
```

### Provider Initialization Flow

1. `main.go` creates the provider server using `providerserver.Serve()`
2. `provider.go` `Configure()` method:
   - Reads configuration (endpoint, token) from HCL or environment variables
   - Creates separate HTTP clients for each resource type (groups, knowledge, models, users)
   - Stores clients in a map passed to resources and data sources
3. Resources and data sources retrieve their specific client from the provider's client map

### Resource/Data Source Pattern

Each resource and data source follows this pattern:
- Implements Terraform Plugin Framework interfaces (`resource.Resource` or `datasource.DataSource`)
- Defines a schema using `schema.Schema` with typed attributes
- Uses a model struct (typically named `*Model`) to map Terraform state to Go structs
- Calls the appropriate client methods (Create, Read, Update, Delete for resources; Read for data sources)
- Transforms between Terraform types (`types.String`, `types.List`, etc.) and client types

### Client Architecture

Each client in `internal/provider/client/*`:
- **client.go**: Implements CRUD operations using standard `net/http`
- **types.go**: Defines Go structs matching OpenWebUI API response/request formats
- Uses Bearer token authentication via `Authorization` header
- Makes requests to `/api/v1/<resource>/` endpoints

### Permission Model (Groups)

Groups have a complex nested permissions structure with four categories:
- **Workspace**: Controls access to models, knowledge, prompts, tools
- **Chat**: Controls chat functionality (file upload, delete, edit, temporary chats, etc.)
- **Sharing**: Controls ability to make resources public
- **Features**: Controls access to advanced features (web search, image generation, etc.)

This maps to nested Terraform schema blocks and requires careful transformation between Terraform's `types.Object` and the client's Go structs.

## Key Conventions

- Use Terraform Plugin Framework (not SDKv2)
- Copyright headers use MIT license and Coalition, Inc (managed via copywrite tool)
- Resource IDs are UUIDs generated by OpenWebUI API
- All API responses use JSON
- Environment variable prefix: `OPENWEBUI_`
- Provider address in registry: `registry.terraform.io/insight2profit/openwebui`

## Important Notes

- The provider uses protocol version 6.0 (see terraform-registry-manifest.json)
- Goreleaser handles multi-platform builds and releases
- Documentation is auto-generated using terraform-plugin-docs (stored in docs/)
- Examples directory contains working Terraform configurations for each resource type
